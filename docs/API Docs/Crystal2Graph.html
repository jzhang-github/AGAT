Crystal2Graph

class CrystalGraph(object)

Read structural file and return a graph.

Code example:

from Crystal2raph import CrystalGraph
cg = CrystalGraph(cutoff = 6.0, mode_of_NN='distance', adsorbate=True)
cg.get_graph('POSCAR')
__init__(self, cutoff=6.0, mode_of_NN='distance', adsorbate=True)
Parameters:

  • cutoff (float) – Cutoff distance when calculate connections of a crystal.

  • mode_of_NN (str) – Mode of how to get the neighbors, which can be: 'voronoi': consider Voronoi neighbors only. 'pymatgen_dist': build graph based on a constant distance using ase module. 'ase_dist': build graph based on a constant distance using pymatgen module. 'ase_natural_cutoffs': build graph from ase which has a dynamic cutoff scheme. In this case, the cutoff is deprecated because ase will use the dynamic cutoffs in ase.neighborlist.natural_cutoffs().

  • adsorbate (bool) – Identify the adsorbate or not.

class ReadGraphs(CrystalGraph)

This object is used to build a list of graphs. Inherited from CrystalGraph. Property (or label) of each graph are loaded from csv file when build new graphs from scratch. On the other hand, they will be loaded from the DGL binary file if all the graphs are loaded from the binary file.

Code example:

from modules.Crystal2Graph import ReadGraphs
import os
graph_reader = ReadGraphs('fname_prop.csv', # csv file generated above.
                          'dataset', # directory contain all frames
                          cutoff       = None, # We don't need this for 'ase_natural_cutoffs'.
                          mode_of_NN   = 'ase_natural_cutoffs', # identify connection between atoms with 'ase_natural_cutoffs'
                          from_binary  = False, # read from structural files
                          num_of_cores = 8, # run parallelly with 8 cores.
                          super_cell   = False) # do not repeat cell for small supercells.

graph_list, graph_labels = graph_reader.read_all_graphs(scale_prop=False, # do not rescale the label.
                                                        ckpt_path='.') # save the information of how to build the graphs.

train_index, validation_index, test_index = TrainValTestSplit(
    0.15, # validation_size: int or float. int: number of samples of the validation set. float: portion of samples of the validation set.
    0.15, # test_size: int or float. int: number of samples of the validation set. float: portion of samples of the validation set.
    os.path.join('files', 'dataset', 'fname_prop.csv'), # csv_file: str. File name of a csv file that contains the filenames of crystals.
    True)() # boolean. Split the dataset by `sklearn.model_selection.train_test_split` or loaded from previously saved txt files.
__init__(self, csv_file, dataset_path, cutoff=6.0, mode_of_NN='distance', from_binary=False, num_of_cores=1, super_cell=True, include_forces=True, adsorbate=True)
Parameters:

  • csv_file (str) – File name of a csv file that contains the filenames of crystals.

  • dataset_path (str) – Path to the dataset. The folder contains DGL binary graph files or files with cif/VASP formate.

  • cutoff (float) – Cutoff distance when calculate connections of a crystal.

  • mode_of_NN (str) – Mode of how to get the neighbors, which can be: 'voronoi': consider Voronoi neighbors only. 'pymatgen_dist': build graph based on a constant distance using ase module. 'ase_dist': build graph based on a constant distance using pymatgen module. 'ase_natural_cutoffs': build graph from ase which has a dynamic cutoff scheme. In this case, the cutoff is deprecated because ase will use the dynamic cutoffs in ase.neighborlist.natural_cutoffs().

  • from_binary (bool) – Whether the graphs are read from scratch or DGL binary graph.

  • num_of_cores (int) – Number of cores when reading graphs from *cif or VASP file. Read graphs from scratch takes much more time than that from binary file. So I recommend reading *cif or VASP file with multiple CPU cores.

  • super_cell (bool) – Repeat small cells or not.

  • include_forces (bool) – Store atomic forces on graph or not.
